# Customer-Shopping-Behavior-Analysis
Customer Shopping Behavior Analysis
Project Overview

Customer Shopping Behavior Analysis is a real-world data analytics project focused on understanding customer purchasing patterns and extracting meaningful business insights from retail data. The goal of this project is to help businesses improve customer retention, optimize marketing strategies, increase revenue, and make data-driven decisions.

This project covers the complete analytics workflow including data cleaning, preprocessing, SQL analysis, exploratory data analysis (EDA), customer segmentation, and interactive dashboard development using Power BI.

Business Problem

Retail businesses often struggle to understand:

Which products generate the highest revenue
Why customer retention is low
How discounts impact purchasing behavior
Which customers are loyal and likely to return
How shipping preferences affect sales
Which customer segments contribute most to profit

This project solves these challenges by analyzing customer shopping behavior and transforming raw data into actionable insights.

Project Objectives
Identify top-performing product categories
Analyze customer retention and repeat purchases
Study discount effectiveness on sales
Compare Standard vs Express shipping behavior
Understand customer segmentation (new, returning, loyal)
Analyze gender-wise and age-wise revenue contribution
Evaluate subscription impact on customer loyalty
Identify seasonal trends and demand patterns
Dataset Information
Source: Kaggle
Dataset Type: Retail / E-commerce Customer Shopping Dataset
Records Included: Customer transactions, product categories, purchase amounts, shipping type, subscription status, discounts, gender, age, and more.

Project Workflow
1. Data Collection

Collected the dataset from Kaggle containing customer shopping and transaction details.

2. Data Cleaning

Handled missing values, removed duplicates, corrected inconsistent values, and prepared clean structured data.

3. SQL Analysis

Used SQL queries to extract meaningful business insights and perform data validation.

4. Exploratory Data Analysis (EDA)

Performed deep analysis to identify trends, patterns, customer behavior, and business opportunities.

5. Customer Segmentation

Grouped customers based on behavior such as repeat buyers, high-value customers, and loyal customers.

6. Dashboard Development

Created interactive Power BI dashboards for business reporting and decision-making.

Key Insights Generated
Top revenue-generating product categories were identified
Returning customers contributed significantly to total revenue
Discounts increased purchase frequency but affected profit margins
